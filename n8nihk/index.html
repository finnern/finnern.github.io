⚡ 2. Lokales Setup: n8n in Docker

Wir nutzen Docker, um eine stabile, isolierte Umgebung zu schaffen, die IHK-Sicherheitshürden umgeht.

  1. Drücken Sie Windows + R, geben Sie cmd ein und drücken Sie Enter.
  2. Führen Sie diesen Befehl aus, um n8n zu starten:
docker run -d --name n8n -p 5678:5678 -v n8n_data:/home/node/.n8n docker.n8n.io/n8nio/n8n

Dashboard öffnen: http://localhost:5678

🧠 3. On-Premise KI: Ollama & Phi-3.5

Intelligente Automatisierung ohne Cloud-Zwang. Wir betreiben unsere KI lokal.

Schritt A: Software beziehen

Ollama Download

Schritt B: Modell laden (in CMD eingeben)

ollama pull phi3.5

Schritt C: Netzwerk-Bypass (Wichtig!)

Beenden Sie Ollama (Taskleiste -> Quit) und starten Sie es in der CMD neu, damit Docker zugreifen darf:

set OLLAMA_HOST=0.0.0.0 && ollama serve

☁️ 4. Microsoft 365 (Azure App Registration)

Verknüpfen Sie n8n mit Ihrem Unternehmens-Ökosystem (Outlook, Teams, Excel).

1. Portal: aka.ms/appregistrations.
2. Redirect URL: Kopieren Sie die OAuth URL aus n8n nach Azure (Typ: Web).
3. Secret: Erstellen Sie einen Schlüssel unter "Zertifikate & Geheimnisse".
ACHTUNG: Kopieren Sie den WERT (Value), nicht die ID!

Sicherheits-Checkliste
Keine Cloud-Abhängigkeit (Self-Hosted).
Datenhoheit gewahrt (Sensible Daten bleiben intern).

Enterprise Best Practices
Fehler 1 vermeiden: Zu großes erstes Projekt – klein anfangen!
Fehler 2 vermeiden: Credentials falsch – API-Keys prüfen (Value vs ID).
Fehler 3 vermeiden: Kein Testen vor Live-Einsatz – immer validieren!
