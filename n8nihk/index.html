⚡ 1. Infrastruktur: n8n in Docker

Die n8n Desktop-App ist veraltet. Wir nutzen Docker für maximale Stabilität.

  1. Drücken Sie Windows-Taste + R, geben Sie cmd ein und drücken Sie Enter.
  2. Kopieren Sie den folgenden Befehl und führen Sie ihn in der schwarzen Box aus:
docker run -d --name n8n -p 5678:5678 -v n8n_data:/home/node/.n8n docker.n8n.io/n8nio/n8n
Troubleshooting: Falls Fehlermeldung "Conflict" erscheint, ist n8n bereits geladen. Nutzen Sie:
docker rm -f n8n (Danach den oberen Befehl erneut ausführen).

Dashboard: http://localhost:5678

🧠 2. Das Gehirn: Ollama & Phi-3.5

Wir nutzen Microsofts Phi-3.5 (3.8 Mrd. Parameter, 2.2 GB), da es lokal extrem performant ist.

A) Modell herunterladen: ollama pull phi3.5 B) Netzwerk-Bypass (WICHTIG!):

Damit Docker mit Windows sprechen darf, muss Ollama im "Server-Modus" mit freigegebenem Host laufen. Beenden Sie Ollama komplett (Taskleiste -> Quit) und tippen Sie in die CMD:

set OLLAMA_HOST=0.0.0.0 && ollama serve
